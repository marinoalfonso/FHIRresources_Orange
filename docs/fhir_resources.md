# FHIR Resources

## What is FHIR?

**FHIR (Fast Healthcare Interoperability Resources)** is an international standard developed by HL7 for exchanging healthcare information electronically. It defines a set of "resources" — structured data models — each representing a specific clinical or administrative concept (patient, observation, medication, etc.).

In this project, FHIR R4 JSON files generated by [Synthea](https://synthea.mitre.org/) are used as input. Synthea is an open-source patient population simulator that produces realistic but entirely synthetic medical records.

## Resources Used

### Patient

The `Patient` resource contains demographic and administrative information about an individual receiving healthcare.

Key fields extracted in `FHIRinputPatient.py`:

| Field | FHIR Path | Description |
|---|---|---|
| Patient ID | `id` | Unique identifier |
| Gender | `gender` | `male` / `female` / `other` / `unknown` |
| Birth date | `birthDate` | Date of birth (YYYY-MM-DD) |
| Marital status | `maritalStatus.coding[].code` | e.g. `M` (married), `S` (single) |
| City | `address[].city` | City of residence |
| State | `address[].state` | State of residence |
| QALY | Extension | Quality-Adjusted Life Year (Synthea extension) |
| DALY | Extension | Disability-Adjusted Life Year (Synthea extension) |

---

### Observation

The `Observation` resource represents measurements, clinical findings, and assessments.

Key fields extracted in `FHIRinputObservation.py`:

| Field | FHIR Path | Description |
|---|---|---|
| Observation ID | `id` | Unique identifier |
| Patient reference | `subject.reference` | Link to the Patient resource |
| Date | `effectiveDateTime` | Date of the observation |
| Code | `code.coding[].code` | LOINC code identifying the observation type |
| Display name | `code.coding[].display` | Human-readable observation name |
| Value | `valueQuantity.value` | Numeric measurement value |
| Unit | `valueQuantity.unit` | Unit of measurement |
| Value (coded) | `valueCodeableConcept.coding[].display` | For categorical observations (e.g. survey answers) |

## Data Flow

```
Synthea JSON files
       │
       ▼
fhirCaricamento.py        ← loads raw JSON bundles
       │
       ├──▶ FHIRinputPatient.py      ← extracts Patient fields → Orange Data Table
       │
       └──▶ FHIRinputObservation.py  ← extracts Observation fields → Orange Data Table
                                              │
                                              ▼
                                     Orange built-in widgets
                                     (visualisation, analysis)
```
